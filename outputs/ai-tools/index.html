GitHub Copilot — Features

主要機能 6つ

01 / INLINE COMPLETION
コード補完（Ghost Text）
入力しながらリアルタイムで次のコードを提案。行補完・関数補完・ブロック補完に対応。Tab で受け入れ。
02 / CHAT
Copilot Chat
IDE内のチャットパネルで質問・リファクタ・デバッグを依頼。コードを選択して「/explain」「/fix」等のコマンドが使える。
03 / CLI
Copilot in the CLI
ターミナルでシェルコマンドをAIで生成・解説。「gh copilot suggest」でコマンドを自然言語から生成。
04 / WORKSPACE
Copilot Workspace
GitHub Issue から実装計画を自動作成。ブランチ・PR・コード変更を一括で提案するエージェント機能。
05 / CODE REVIEW
AIコードレビュー
Pull Requestに対してAIが自動レビューコメントを付与。バグ・セキュリティ問題・改善点を指摘。
06 / EXTENSIONS
Copilot Extensions
サードパーティサービスをCopilot Chatに統合する拡張機構。Sentry・Docker・Datadog等と連携可能。

Decision Guide

どれを選ぶか

⬡ Copilot を選ぶ
こんな人・状況に最適
  • 今のIDEを変えたくない
  • コード補完をメインに使いたい
  • チーム全員に同じ環境を配布したい（Business/Enterprise）
  • GitHub のPR・Issue ワークフローで使いたい
  • 企業のセキュリティ要件が厳しい
  • まず無料で試してみたい
◈ Claude Code を選ぶ
こんな人・状況に最適
  • 「〇〇を全部実装して」という大きな指示を出したい
  • 複数ファイルを横断する変更を一括でやってほしい
  • IDE に縛られず、どこでも同じ操作をしたい
  • AIエージェントをカスタム設計・組み合わせたい
  • Claudeの長文理解力（200K）を最大限活用したい
⬢ Cursor を選ぶ
こんな人・状況に最適
  • AI統合IDEに完全移行したい
  • 状況に応じてAIモデルを切り替えたい
  • VSCodeの慣れた環境を保ちつつAIを強化したい
  • コードベース全体を参照しながら会話したい
  • GUIで直感的にAI機能を操作したい
Strategy

競合ではなく
"役割分担"として使う

3つのツールは競合ではなく、それぞれが異なる「仕事の粒度」を担う。 目的ごとに最適なツールを使い分け、あるいは組み合わせることで 開発速度と品質が最大化される。

Step 1
基盤環境のセットアップ
1
パッケージマネージャーを導入
# Mac
/bin/bash -c "$(curl -fsSL https://brew.sh/install.sh)"
# Windows → WSL2 を先にセットアップ
2
Node.js 20 LTS をインストール（nvm推奨）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
nvm install 20 && nvm use 20
3
Python 3.11 をインストール
brew install pyenv
pyenv install 3.11 && pyenv global 3.11
4
Gitの初期設定
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
💡 Windowsを使う場合はWSL2（Ubuntu）環境上での作業を強く推奨。
パス区切り・権限・シェルの差異でトラブルになりやすい
バージョン確認コマンド
node -v # v20.x.x 以上
python3 --version # 3.11.x 以上
git --version # git version 2.x
npm -v # 10.x 以上

Step 3
MCPサーバー 接続設定
🔒 このファイルには機密情報（APIキー）が含まれます。 Gitに含めない・他者に共有しない・クラウドに平文保存しない
~/.claude/claude.json の構造
{
  "mcpServers": {
    "chatwork": {
      "command": "npx",
      "args": ["@chatwork/mcp-server"],
      "env": {
        "CHATWORK_API_TOKEN": "your-token-here"
      }
    },
    "notion": { /* 同様の構造 */ }
  }
}
接続サービス一覧
サービス 必要なもの 優先度
Gmail / Google Drive OAuth2 認証（ブラウザ承認） 必須
Chatwork APIトークン（設定画面で発行） 必須
Notion Integration Token（notion.so） 必須
Slack Bot Token（OAuth設定） 推奨
Obsidian 直接ファイル書き込みで代替可 任意
📌 接続確認は claude 起動後、自然言語でツールを呼び出してテスト

Troubleshooting
よくあるトラブル Q&A
🔴 Problem 1

MCPサーバーに接続できない
（ツール呼び出しがエラーになる）

✅ Solution

① APIキー・Tokenの値が正しいか確認
② サービス側でIntegration/権限が承認されているか確認
③ claude.json のパスに誤字がないか確認

🔴 Problem 2

ターミナルで
claude: command not found と出る

✅ Solution

① npm global の PATH が通っているか確認
② export PATH="$PATH:$(npm prefix -g)/bin" を .zshrc に追加
③ ターミナルを再起動して再試行

🔴 Problem 3

エージェントが指示通りに動かない
（別の動作をしてしまう）

✅ Solution

① CLAUDE.md の絶対パスが移植先と合っているか確認
② agents/ 配下のファイルが正しい場所にあるか確認
③ claude を完全終了して再起動

💬 解決しない場合は KISHINO MASASHI まで状況・エラーメッセージをそのままスクリーンショットで共有してください
